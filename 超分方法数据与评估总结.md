# 超分方法数据处理、合成与评估总结

## 1. LSDIR: A Large Scale Dataset for Image Restoration (CVPRW 2023)

只有数据集，提出的就是数据集，没有训练部分

数学方法进行初步过滤：拉普拉斯算子，平坦区域，人工

后处理：LSDIR X4：3840 × 2160 -> 960 × 540

### 数据处理
- **数据来源**：从Flickr自动爬取4K高分辨率图像，使用超过20,000个关键词（来自Flickr2K、Flickr热门标签、ImageNet 1K、ImageNet 21K标签集）进行搜索
- **模糊检测**：使用拉普拉斯方差（Variance of Laplacian）作为模糊分数，仅保留分数在[150, 8000]范围内的图像，排除过于模糊或噪声过大的图像
- **平坦区域检测**：将图像划分为240×240非重叠patch，使用Sobel算子计算边缘丰富度，方差低于800的patch视为平坦区域；超过50%平坦patch的图像被剔除
- **下采样降噪**：原始图像通过Lanczos重采样下采样4倍（LSDIR X4）来抑制噪声和JPEG压缩伪影
- **人工质量检查**：两轮人工标注——第一轮粗筛（删除>70%图像），第二轮精细筛选去除假阳性，仅保留锐利、无模糊无噪声的高质量图像

### 数据合成
- **LR-HR配对合成**：通过经典的降质模型合成低分辨率图像，包括双三次下采样（bicubic downsampling）、不同降质类型等
- 最终数据集包含 **84,991张训练图像、1,000张验证图像、1,000张测试图像**

### 评估方法
- **全参考指标**：PSNR、SSIM
- **测试集**：Set5、Set14、BSD100、Urban100、Manga109、DIV2K Val
- **任务覆盖**：超分辨率（×2/×3/×4）、去噪、JPEG去块效应、去模糊、去马赛克、真实世界SR
- **对比网络**：EDSR、SwinIR、MSRResNet、IMDN等

## 2. SUPIR: Scaling Up to Excellence (CVPR 2024)

基于controlnet输入lr图片，引入了语言模型，支持通过文本提示对修复过程进行灵活控制

超过 **2000万张** 1024×1024高质量、纹理丰富的高分辨率图像，每张配有描述性文本标注，70K张来自FFHQ-raw的不对齐高分辨率人脸图像，以及100K张由SDXL生成的负质量样本图像

### 数据处理

- **数据收集**：收集了超过 **2000万张** 1024×1024高质量、纹理丰富的高分辨率图像，每张配有描述性文本标注
- **额外数据**：70K张来自FFHQ-raw的不对齐高分辨率人脸图像，以及100K张由SDXL生成的负质量样本图像（对应负质量提示词）
- **文本标注**：使用13B参数的多模态语言模型（LLaVA）为图像生成内容描述作为提示词
- **负质量样本**：用SDXL生成100K低质量图像（对应如"oil painting, cartoon, blur, dirty, messy, low quality"等负质量描述词），加入训练数据以使模型学习负质量概念

### 数据合成
- **降质模型**：使用Real-ESRGAN降质管线合成LQ图像，训练时裁剪为512×512 patch
- **正负提示词**：正提示词描述图像质量和内容，负提示词描述低质量特征，结合CFG（Classifier-Free Guidance）

### 评估方法
- **全参考指标**：PSNR、SSIM、LPIPS
- **无参考指标**：ManIQA、ClipIQA、MUSIQ
- **合成数据测试**：单降质（SR×4、SR×8）和混合降质（Blur+SR、SR+Noise、Blur+SR+Noise+JPEG）
- **真实世界测试**：60张来自RealSR、DRealSR、Real47及网络的LQ图像
- **用户研究**：20名参与者的偏好评估
- **核心发现**：SUPIR在无参考指标上全面领先，但全参考指标（PSNR/SSIM）偏低，存在感知质量与保真度指标的失配问题

## 3. LucidFlux: Caption-Free Universal Image Restoration (ICLR 2026)

### 数据处理
- **数据来源**：从Pexels和Unsplash通过官方API收集2.3M图像（使用23,914个关键词），加上Photo-Concept-Bucket的557K图像，共2.9M候选图像
- **三阶段自动过滤管线**：
  1. **模糊检测**：拉普拉斯方差在[150, 8000]范围内保留
  2. **平坦区域检测**：240×240 patch的Sobel方差低于800视为平坦，超过50%平坦patch的图像剔除
  3. **IQA过滤**：使用CLIP-IQA排序，仅保留前20%（257K张）
- **最终数据集**：257K + 84K（LSDIR高质量样本）= **342K高质量图像**
- **配对生成**：使用Real-ESRGAN降质管线，4个epoch，共产生 **1.36M图像对**

### 数据合成
- **降质管线**：Real-ESRGAN降质管线（与DreamClear实现一致）
- **轻量修复代理**：使用SwinIR作为LRP（Lightly Restored Proxy），为双分支条件器提供输入
- **无需文本标注**：用SigLIP特征替代VLM生成的caption，避免caption偏差

### 评估方法
- **无参考指标**：CLIP-IQA+、Q-Align、MUSIQ、MANIQA、NIMA、CLIP-IQA、NIQE
- **全参考指标**：PSNR、SSIM、LPIPS（仅在合成数据上）
- **测试集**：
  - 合成：DIV2K-Val、LSDIR-Val
  - 真实：RealSR、DRealSR、RealLQ250
- **对比方法**：ResShift、StableSR、SinSR、DiffBIR、SeeSR、DreamClear、SUPIR
- **商业模型对比**：HYPIR-FLUX、Seedream 4.0、Topaz、Gemini-NanoBanana、MeiTu SR
- **消融实验**：双分支条件器、SigLIP对齐、时间步与层自适应调制（TLCM）、数据规模

## 4. VOSR: A Vision-Only Generative Model for Image Super-Resolution (arXiv 2026)

### 数据处理
- **数据收集**：从网络收集图像，经质量与多样性过滤（梯度过滤、图像熵过滤、分辨率过滤、类别级去重与平衡），获得约 **100M图像**
- **纯视觉训练**：不使用任何多模态文本-图像监督，仅使用合成的LR-HR配对和视觉编码器辅助特征

### 数据合成
- **降质管线**：Real-ESRGAN降质管线合成LR-HR训练对
- **条件设计**：
  - **结构条件**：用VAE编码器编码LR图像为潜在空间结构条件（cstr）
  - **语义条件**：用预训练视觉编码器（DINOv2）提取视觉语义特征（csem）
- **引导策略**：提出面向修复的引导（Restoration-Oriented Guidance），用部分条件分支（保留弱LR结构、移除语义）替代标准CFG的无条件分支

### 评估方法
- **全参考指标**：PSNR、SSIM（Y通道）、LPIPS、DISTS、AFINE-FR
- **无参考指标**：NIQE、MUSIQ、MANIQA、AFINE-NR、TOPIQ-NR
- **测试集**：
  - 合成：LSDIR-Val（250张）
  - 真实：RealSR、新提出的ScreenSR（130对，通过屏幕重摄影管线构建）
- **用户研究**：10名志愿者，评估感知质量与LR输入一致性
- **对比方法**：ResShift、StableSR、PASD、SeeSR、DiT4SR（多步）；SinSR、OSEDiff、PiSA-SR（单步）
- **核心发现**：纯视觉模型在感知质量上可与T2I方法竞争，训练成本仅约1/10

## 5. Z-Image: An Efficient Image Generation Foundation Model (arXiv 2025)

### 数据处理
- **高效数据基础设施**，包含四个模块：
  1. **数据画像引擎（Data Profiling Engine）**：提取多维特征（元数据、清晰度、异常检测、文本描述等）
  2. **跨模态向量引擎（Cross-modal Vector Engine）**：基于数十亿嵌入进行语义去重和定向检索
  3. **世界知识拓扑图（World Knowledge Topological Graph）**：层次化组织知识，识别欠表示概念
  4. **主动策展引擎（Active Curation Engine）**：闭环数据标注，自动采样识别薄弱概念
- **技术质量评估**：压缩伪影检测、模糊检测、人脸/文本检测等

### 数据合成
- **纯真实数据训练**：不蒸馏其他模型的合成数据
- **渐进式训练课程**：
  1. 低分辨率预训练（256²）
  2. 全能预训练（Omni-pretraining）：统一多任务（任意分辨率生成、文生图、图编辑）
  3. PE-aware监督微调
- **编辑数据对**：通过图形化表示高效构建图像编辑对

### 评估方法
- **人类偏好评估**：Elo评级系统，与Flux 2 dev对比
- **定量评估**：T2I生成（FID、CLIP等）、指令驱动图像编辑
- **对比模型**：Lumina-Image 2.0、Qwen-Image、Seedream 3.0/4.0、HunyuanImage3.0、Imagen 4 Ultra、FLUX 2、Nano Banana Pro
- **重点能力**：真实感生成、中英双语文字渲染、指令跟随编辑
- **总训练成本**：314K H800 GPU小时（约$628K）

## 6. Boosting Latent Diffusion with Flow Matching (arXiv 2024)

基于flow，有模型的训练，数据工程过程如下：

```markdown
1. 原始数据源（原料采购）
论文并没有自己去拍照片，而是使用了业内公认的高质量“原料库”：FacesHQ（人脸）：结合了 CelebA-HQ 和 FFHQ。LHQ（风景）：9 万张高清风景图。Unsplash（通用）：高质量、多样化的真实图像。这些数据保证了模型能够见多识广，从细腻的人脸毛孔到壮丽的山川纹理都能处理。
2. 潜空间对齐（数据建模）
这是最关键的一步。模型不是在像素级别工作的，而是在潜空间。$x_1$（目标高清码）：直接把真实高清原图扔进预训练好的 VAE Encoder，得到一个“高清潜码”。$x_0$（输入起点码）：为了得到和 $x_1$ 对应的低画质起点，作者对图像做了降采样。但重点来了，为了让 $x_0$ 和 $x_1$ 维度一致以便进行流匹配（Flow Matching），他们必须对 $x_0$ 进行上采样。
3. 核心黑科技：像素空间上采样（PSU）
在数据工程中，作者通过对比实验发现了一个重要细节：如果直接在潜空间（Latent Space）用双线性插值（Bilinear）把 $x_0$ 拉大，会产生很多恼人的伪影（Artifacts）。论文采用的 PSU 流程如下：解码：将低分辨率的潜码变回像素图像。拉伸：在像素层级进行高质量的上采样。再编码：重新编码回潜空间。这样处理出来的数据起点 $x_0$ 非常干净，没有杂乱的数学伪影，模型学起来事半功倍。
4. 退化模拟（模拟极端环境）
为了让模型更耐造，论文在数据工程中引入了 Real-ESRGAN 的退化方案。处理手段：在生成训练用的低质图像时，随机加入高斯模糊、传感器噪声、JPEG 压缩损失。目的：这相当于在训练 Java 异常处理（Exception Handling）时，故意抛出各种空指针、连接超时异常，让你的代码（模型）在真实世界面对烂画质时依然能稳健运行。
5. 噪声增强（数据稳定器）
在训练对构建的最后一步，作者在 $x_0$ 上添加了一个特定强度的高斯噪声（文中实验得出 $t=400$ 时效果最好）。逻辑分析：这在数学上叫“平滑概率密度路径”。通俗地说，就是给起点的分布加一点点扰动，防止模型“死记硬背”特定的像素点，从而让它在推理时对各种细微的画质差异都有更好的泛化能力。
```



### 数据处理

- **训练数据**：Unsplash数据集（通用T2I任务）、FacesHQ（CelebAHQ+FFHQ，高分辨率合成）、LHQ（90K高分辨率风景图）
- **评估数据**：LAION-5B高分辨率子集、COCO数据集

### 数据合成
- **Coupling Flow Matching (CFM)**：在潜在空间中，从低分辨率潜在编码到高分辨率潜在编码的最优传输映射
- **噪声增强**：对低分辨率潜在编码添加余弦调度的高斯噪声（最佳噪声步数t=400），平滑基础概率密度
- **上采样方法**：像素空间上采样（PSU）效果最佳——先解码到像素空间，双线性上采样后重新编码
- **架构**：小扩散模型生成低分辨率内容 → CFM模块映射到高分辨率潜在空间 → VAE解码器输出高分辨率图像

### 评估方法
- **生成质量指标**：FID、Patch-FID（pFID）、CLIP Score
- **超分辨率指标**：PSNR、SSIM
- **推理速度**：每张图像推理时间（s/im）
- **对比方法**：
  - T2I合成：LCM-LoRA SDXL、SDXL、StableCascade、CogView3
  - 超分辨率：BSRGAN、RealERSGAN、DASR、LDM、StableSR、ResShift
- **测试集**：COCO-5k（1024²）、LAION-5k、DIV2K、RealSR
- **核心优势**：CFM轨迹更直，训练更快收敛，推理步数更少；可在低NFE下超越StableSR
